## 📅 日期

## 🎯 方案名称
混合模式（云端为主，本地备用 + 下载功能）

## 📋 方案概述
- **云端**：阿里云存储大部分经文（近万本）
- **本地**：只保留热门经典备用 + 目录索引
- **下载功能**：用户可选择下载到本地，离线阅读
- **特点**：APP体积小、可在线更新、有备用保障、支持离线

---

## 🏗️ 架构设计

### 数据流向
```
用户请求 → 检查本地缓存 → 有 → 显示
                    ↓ 没有
                  从阿里云下载 → 显示 → 缓存到本地
                    ↓ 失败
                  用本地备用数据
```

### 文件分布
| 位置 | 文件 | 内容 | 说明 |
|-----|------|------|-----|
| 本地 | static/data/daojing.json | 道藏目录 | 保留 |
| 本地 | static/data/fojing.json | 大藏经目录 | 保留 |
| 本地 | static/data/articles.json | 热门经典内容 | 只留几本 |
| 阿里云 | daojing_list.json | 完整道藏目录 | 在线 |
| 阿里云 | fojing_list.json | 完整大藏经目录 | 在线 |
| 阿里云 | articles/ | 所有经文内容 | 在线 |

## 📱 下载功能设计

### 按钮位置
- ✅ 书籍列表页：每本书旁边显示[下载]/[删除]按钮
- ✅ 阅读页面：工具栏添加下载/删除按钮

### 交互流程
**下载流程：**
1. 用户点击[下载]按钮
2. 显示"正在下载..."提示
3. 下载完成后显示"下载完成"提示
4. 按钮变为[删除]

**删除流程：**
1. 用户点击[删除]按钮
2. 弹框确认："确定要删除这本经书吗？"
3. 用户点确定 → 删除成功，按钮变回[下载]
4. 用户点取消 → 不删除

### 缓存存储
- 使用 uni.storage 存储下载的经文内容
- 单独存储一个"已下载列表"用于管理

## 📁 云端部署指南（给小白看）

### 阿里云后台操作步骤
1. 登录阿里云云开发后台
2. 找到云存储/云数据库
3. 上传 JSON 文件
4. 复制文件访问地址
5. 填到 config/yunpeizhi.js 里

### 添加新经文步骤
1. 打开阿里云后台
2. 找到对应的 JSON 文件
3. 按格式添加新经文
4. 保存，完成！
