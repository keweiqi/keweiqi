## 简介

本应用采用 **本地数据 + 云端支持** 的混合模式：
- 默认使用本地数据，稳定可靠
- 后期可以通过云端更新数据，无需重新写代码
- 小白用户完全可以自己操作！

## 本地更新数据（最简单）

### 适用场景
- 你想更新诗词、儒教等内容
- 不需要频繁发布新版本
- 不需要让所有用户立刻看到新内容

### 操作步骤

#### 第1步：找到数据文件
在项目的 `static/data/` 文件夹里，有4个数据文件：
- `shici.js` - 诗词经典数据
- `rujiao.js` - 儒教典籍数据
- `jingdian.js` - 道藏、大藏经数据（已有）

#### 第2步：编辑数据文件
打开对应的 `.js` 文件，直接修改内容即可：

**例子：修改一首诗**
```javascript
// 原来的
{ name: '静夜思', displayName: '静夜思', desc: '李白作' }

// 修改后
{ name: '静夜思', displayName: '静夜思（新版）', desc: '李白经典名作' }
```

**例子：新增一首诗**
```javascript
// 在 tangshi 数组里加一个新对象
{ name: '黄鹤楼送孟浩然之广陵', displayName: '黄鹤楼送孟浩然之广陵', desc: '李白作，故人西辞黄鹤楼' }
```

#### 第3步：添加经文内容（可选）
如果是新增的内容，需要在 `shiciContent`、`rujiaoContent` 里也加上对应的内容。

**例子：**
```javascript
's黄鹤楼送孟浩然之广陵': '故人西辞黄鹤楼，烟花三月下扬州。\n孤帆远影碧空尽，唯见长江天际流。',
```

#### 第4步：重新打包 APP
修改完后，重新打包 APP 发布，用户更新 APP 后就能看到新内容！

## 云端更新数据（高级）

### 适用场景
- 你想让用户不更新 APP 就能看到新内容
- 需要频繁更新数据
- 内容很多，不想每次都重新打包

### 前提准备
你需要先有一个 **阿里云 OSS**（对象存储服务）账号，或者其他类似的云存储服务。

### 操作步骤

#### 第1步：准备数据文件
把你要上传的数据文件（比如 `shici.js`、`rujiao.js`）单独保存，去掉 `export` 语句，保存为 `.json` 格式。

**例子：shici.json**
```json
{
  "shiciBooks": {
    "tangshi": [...],
    "songci": [...],
    "shijing": [...]
  },
  "shiciContent": {...}
}
```

#### 第2步：上传文件到云端
1. 登录阿里云 OSS 控制台
2. 上传刚才准备好的 `.json` 文件
3. 复制文件的 **访问地址（URL）**

#### 第3步：修改配置文件
打开 `config/yunpeizhi.js` 文件，找到相应的配置项，填上刚才复制的地址：

```javascript
const peizhi = {
  qiyongyunduan: true,  // 把这个改成 true！

  ali: {
    daojing_list: '',      // 道藏目录地址
    fojing_list: '',       // 大藏经目录地址
    shici_list: 'https://你的阿里云地址/shici.json',  // ← 填这里
    rujiao_list: 'https://你的阿里云地址/rujiao.json', // ← 填这里
    jingwen_qianzhui: ''
  },
  chaoshi: 5000
}
```

#### 第4步：发布 APP（首次）
第一次使用云端功能，需要重新打包并发布 APP。

#### 第5步：以后更新数据
以后要更新内容时：
1. 修改数据文件
2. 重新上传到云端，覆盖旧文件
3. 用户打开 APP，自动下载新数据！

**不需要重新打包，不需要用户更新 APP！**

## 常见问题解答

### Q1: 我完全不懂代码，能操作吗？
**A: 可以！**
- 本地更新只需要用记事本打开文件修改文字
- 云端更新只需要上传文件和复制粘贴地址
- 代码部分已经写好了，你不需要碰

### Q2: 云端更新有什么好处？
**A:**
- 用户不需要更新 APP 就能看到新内容
- 你可以随时更新，很方便
- 文件可以放很多内容，APP 不会变大

### Q3: 云端和本地能一起用吗？
**A: 可以！**
- 默认用本地数据
- 你可以随时切换到云端，或者改回来
- 云端加载失败时，APP 会自动用本地数据

### Q4: 上传文件到阿里云要钱吗？
**A: 阿里云 OSS 有免费额度，初期应该够用。** 具体看阿里云的价格政策。

### Q5: 我搞砸了怎么办？
**A: 没关系！**
- 本地备份一份原始文件
- 云端操作错了可以重新上传
- 实在不行，把 `qiyongyunduan` 改回 `false`，就只用本地数据了
